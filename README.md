## 一、搬瓦工 DC6 CN2 GIA-E 是个什么东西

先把名字拆开讲。搬瓦工的英文商标是 BandwagonHost，背后是美国 IT7 Networks，从 2004 年就在做主机生意。DC6 是它的一个机房代号，官方编号 USCA\_6，物理位置在美国洛杉矶。

重点在后面那个尾巴——**CN2 GIA-E**。这是中国电信的高端优化线路，全称 Global Internet Access - ECommerce，翻译过来就是"电商级全球互联网接入"。搬瓦工官网对 CN2 GIA 的解释很直白：这是中国方向数据传输最贵的一种方式，每兆带宽单价能到 120 美元/Mbps，容量还非常有限，但对跨境访问的稳定性最好。

CN2 GIA-E 比 CN2 GIA 多了个"E"——ECommerce。搬瓦工把 CN2 GIA-E 定位为电商优化线路，意思是国际段线路也做了优化，比纯 CN2 GIA 在国际回程上更稳一点。普通洛杉矶机房走的是 AS4134（ChinaNet/163 网），晚高峰经常拥堵到丢包 30% 以上；DC6 走的是 AS4809（CN2 GIA），线路容量小但稳，拥堵和丢包控制得明显更好。

所以 DC6 的卖点从来不是"配置多猛"，而是"线路多稳"。它适合的是把稳定性放在价格前面的那拨人。

## 二、DC6 三网优化线路说明

进 KiwiVM 后台看 DC6 机房的名字，会发现一串缩写：**CT CN2GIA-E, CMIN2, CUP**。这三个缩写分别对应三家运营商方向的优化标识：

- **CT CN2GIA-E**：中国电信方向的 CN2 GIA-E 优化线路（AS4809）
- **CMIN2**：中国移动方向的高端优化线路（AS58807）
- **CUP**：中国联通方向的优化线路（AS10099）

也就是说，DC6 不是只优化电信一家，而是对三网都做了优化。这也是它和普通洛杉矶机房最核心的区别——普通机房走普通国际线路，DC6 是明确按"电信 GIA + 移动 CMIN2 + 联通 CUP"三网优化来配的。

当然，"三网优化"不等于"所有人一定满速"。实际表现还是会受你本地宽带、运营商调度、测试时间影响。但比起普通线路，DC6 的线路基础明显更适合中国大陆方向访问。

## 三、搬瓦工 DC6 实测：延迟、速度、丢包率、路由

这部分是搜索"搬瓦工 DC6 测评"的人最想看的。综合多个第三方测评站点的实测数据，DC6 CN2 GIA-E 的表现如下。

### 延迟测试

DC6 三网平均延迟基本压在 200ms 以内，电信线路平均延迟约 161–168ms，最低能到 133ms——这个数字已经接近中美之间物理距离决定的理论最低延迟了。联通和移动稍高一点，但也都在可接受范围内。

简单对照一下：普通洛杉矶机房晚高峰延迟动辄 250ms+ 还要波动，DC6 日常稳定在 130–180ms 之间，差距是肉眼可见的。

### 下载速度

DC6 CN2 GIA-E 套餐带宽起步 2.5Gbps，高配版本最高 10Gbps。实测国内多地下载速度基本都能跑到 400Mbps 以上，晚高峰也能维持，不像普通线路一到晚上就掉到几十 Mbps。

### 丢包率

DC6 的丢包率测试结果是这份测评里最漂亮的部分。从多个测评站点的数据看，三网丢包率绝大部分地区是 0，少量地区在 1% 以内。对照普通国际线路晚高峰 30%+ 的丢包，DC6 这个表现基本属于"用钱砸出来的稳"。

### 回程路由

回程路由是判断线路真假的关键。从实测 traceroute 看，DC6 三网回程都走 CN2 GIA：

- **电信回程**：洛杉矶 → 上海 CN2 GIA（59.43.x.x）→ 各省落地，全程 AS4809
- **联通回程**：洛杉矶 → 上海 CN2 GIA → 上海联通 AS4837 落地
- **移动回程**：洛杉矶 → 上海 CN2 GIA → 移动 AS9808 落地

注意联通和移动的回程最后一段会切到各自的骨干网，但前半段国际线路都走 CN2 GIA，这就是 DC6 区别于普通机房的地方——普通机房回程经常绕 AS4134 或者直接走普通国际出口，DC6 是国际段全程 CN2 GIA 兜底。
